# v0.9.2 — Interior-only panel HA state mapping

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix every arm/disarm button erroring on `INTERIOR_ONLY` family panels (SDVFAST, SDVFSW). Integration currently targets `PARTIAL/ON` or `TOTAL/ON` regardless of panel, and the resolver correctly rejects perimeter commands on panels with no perimeter sensors — so `UnsupportedCommandError` fires for every action. Also corrects reverse proto mapping: interior-only states `P`/`T` currently render as `ARMED_CUSTOM_BYPASS`; on `INTERIOR_ONLY` panels they are the primary home/away states.

Root cause and web-bundle validation of the fix in the session transcript and [`docs/findings/arm-command-vocabulary.md`](../findings/arm-command-vocabulary.md).

**Reported by:** Issue #3, alan210874 (SDVFAST). Failure block:

```
panel: SDVFAST
family: INTERIOR_ONLY
current_proto: 'D'
active_services: [ARM, ARMDAY, ARMINTFPART, ARMNIGHT, ARMPARTFINT, DARM]
error_type: UnsupportedCommandError
error_message: Panel 'SDVFAST' cannot honour 'ARMDAY1PERI1': missing active service(s): PERI
```

**Architecture (approach A — family-aware entity layer):**
- `_ARM_TARGETS`: `dict[PanelFamily, dict[AlarmControlPanelState, AlarmState]]` — entity picks the target state based on panel family. PERI_CAPABLE keeps `PARTIAL/ON` + `TOTAL/ON` (current). INTERIOR_ONLY collapses to `PARTIAL/OFF` + `TOTAL/OFF`.
- `_STATE_MAP`: `dict[PanelFamily, dict[AlarmState, AlarmControlPanelState]]` — reverse map split by family. INTERIOR_ONLY maps proto `P`/`T` to `ARMED_HOME`/`ARMED_AWAY` (not `CUSTOM_BYPASS`). Perimeter-involving states absent from the INTERIOR_ONLY map by design — a panel returning one violates the family invariant and crashes loud on lookup.
- No changes to `verisure_italy/` (resolver, client, models). The fix lives entirely in the HA entity layer.

**Web-bundle validation** (`/tmp/verisure-web-bundle/main.2.4.2.js`):
- `R(e)` classifier: `SDVFAST`/`SDVFSW` → `false` (INTERIOR_ONLY); matches `PANEL_FAMILIES` exactly.
- Arm resolver `w({alarmMode, currentMode, isCU})`: `ARM_DAY` → `ARMDAY1` (no `PERI`), `ARM_TOTAL` → `ARM1` (no `PERI`). Web UI never emits `ARMDAY1PERI1`/`ARM1PERI1` on INTERIOR_ONLY panels.
- `availableModes` for INTERIOR_ONLY UI: `{DISARM, ARM_DAY, ARM_NIGHT, ARM_TOTAL}` — no perimeter variants offered.

**Tech Stack:** Python 3.12 async, Pydantic v2, pytest, pyright strict, ruff. No new dependencies.

**Out of scope:**
- `ARM_NIGHT` support (no HA UI surface today; separate ticket when a reporter asks).
- Model-layer refactor collapsing `PerimeterMode` for INTERIOR_ONLY panels (approach B in brainstorm). Overkill for a UX bug.
- Resolver / client / core model changes.

```python
# Canonical arm targets per panel family.
_ARM_TARGETS: dict[PanelFamily, dict[AlarmControlPanelState, AlarmState]] = {
    PanelFamily.PERI_CAPABLE: {
        AlarmControlPanelState.ARMED_HOME: AlarmState(
            interior=InteriorMode.PARTIAL, perimeter=PerimeterMode.ON
        ),
        AlarmControlPanelState.ARMED_AWAY: AlarmState(
            interior=InteriorMode.TOTAL, perimeter=PerimeterMode.ON
        ),
    },
    PanelFamily.INTERIOR_ONLY: {
        AlarmControlPanelState.ARMED_HOME: AlarmState(
            interior=InteriorMode.PARTIAL, perimeter=PerimeterMode.OFF
        ),
        AlarmControlPanelState.ARMED_AWAY: AlarmState(
            interior=InteriorMode.TOTAL, perimeter=PerimeterMode.OFF
        ),
    },
}

_DISARMED = AlarmState(interior=InteriorMode.OFF, perimeter=PerimeterMode.OFF)

# Reverse map: AlarmState -> HA state, per family.
# Absent keys raise KeyError — fail-secure on family-impossible states.
_STATE_MAP: dict[PanelFamily, dict[AlarmState, AlarmControlPanelState]] = {
    PanelFamily.PERI_CAPABLE: {
        _DISARMED: AlarmControlPanelState.DISARMED,
        _ARM_TARGETS[PanelFamily.PERI_CAPABLE][AlarmControlPanelState.ARMED_HOME]: AlarmControlPanelState.ARMED_HOME,
        _ARM_TARGETS[PanelFamily.PERI_CAPABLE][AlarmControlPanelState.ARMED_AWAY]: AlarmControlPanelState.ARMED_AWAY,
        PROTO_TO_STATE[ProtoCode.PERIMETER_ONLY]: AlarmControlPanelState.ARMED_CUSTOM_BYPASS,
        PROTO_TO_STATE[ProtoCode.PARTIAL]: AlarmControlPanelState.ARMED_CUSTOM_BYPASS,
        PROTO_TO_STATE[ProtoCode.TOTAL]: AlarmControlPanelState.ARMED_CUSTOM_BYPASS,
    },
    PanelFamily.INTERIOR_ONLY: {
        _DISARMED: AlarmControlPanelState.DISARMED,
        _ARM_TARGETS[PanelFamily.INTERIOR_ONLY][AlarmControlPanelState.ARMED_HOME]: AlarmControlPanelState.ARMED_HOME,
        _ARM_TARGETS[PanelFamily.INTERIOR_ONLY][AlarmControlPanelState.ARMED_AWAY]: AlarmControlPanelState.ARMED_AWAY,
    },
}
```

- [ ] **Step 3: Wire entity to use the family map**

In `VerisureAlarmPanel`:

```python
@property
def _panel_family(self) -> PanelFamily:
    return PANEL_FAMILIES[self.coordinator.installation.panel]

def _update_alarm_state(self) -> None:
    """Update _attr_alarm_state from coordinator data. Crashes on impossible state."""
    self._attr_alarm_state = _STATE_MAP[self._panel_family][self.coordinator.data.alarm_state]
    self._update_supported_features()
    self._update_force_attributes()

async def async_alarm_arm_home(self, code: str | None = None) -> None:
    target = _ARM_TARGETS[self._panel_family][AlarmControlPanelState.ARMED_HOME]
    await self._async_arm(target, AlarmControlPanelState.ARMED_HOME, "armed_home")

async def async_alarm_arm_away(self, code: str | None = None) -> None:
    target = _ARM_TARGETS[self._panel_family][AlarmControlPanelState.ARMED_AWAY]
    await self._async_arm(target, AlarmControlPanelState.ARMED_AWAY, "armed_away")
```

`_DISARMED` usage at line 187 (force-context clearing) stays as-is — disarmed is disarmed in both families.
