## S1: Workflow Bootstrap — start/close skills + docs structure (2026-04-17)

### Skills
- Added `.claude/skills/start/` and `.claude/skills/close/` adapted
  from the `/srv/gastone` skills. Public-safe wording (no blasphemy
  opener). `close` has a mandatory CHANGELOG step and checkpoint
  flush; `start` reports on the review gate, active checkpoint,
  unreleased items, and git state.

### Docs structure
- New repo layout mirroring gastone's (lighter): `docs/checkpoints/`,
  `docs/findings/`, `docs/decisions.md`, `docs/patterns.md`,
  `docs/todo.md`.
- Dropped `docs/superpowers/` — the HA-integration plan/spec it held
  shipped in v0.8.0. Plans and specs are ephemeral; `findings/` and
  `reviews/` are the durable records.
- Migrated four Verisure-specific memories out of user auto-memory
  into `docs/findings/` so they are now committed with the repo:
  `verisure-api.md`, `arm-exception-flow.md`, `cameras.md`,
  `camera-capture-tuning.md`.
- `docs/decisions.md` seeded with the core design tenets (fail-secure,
  two-axis state model, parse-at-boundary, no HA-level PIN, reload
  vs restart, no default arguments).
- `docs/patterns.md` seeded with HA gotchas (sync-in-async deadlock,
  never block the event loop, no `yaml.dump` on `automations.yaml`,
  UTC vs local time, HA 2026.4+ thread safety in timer callbacks,
  entity registry cache, `_last_proto` sync).
- Auto-memory `MEMORY.md` updated to reflect the migration;
  `project_*` memories that moved into the repo are deleted.

### Decisions
- Auto-memory keeps only `user/feedback/reference` types going
  forward. Project-type findings live in-repo so they travel with
  the code and can be reviewed in PRs.
- `CP01` is deliberately thin — pre-history is represented by the
  CHANGELOG rather than backfilling fake session entries.
