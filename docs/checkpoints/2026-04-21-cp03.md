## S1: v0.9.0 Panel-Aware Command Resolver (2026-04-24)

Multi-phase marathon: reverse-engineered Verisure web app bundle to
extract the full `ArmCodeRequest` / `DisarmCodeRequest` enum, decoded
the per-target command resolver, expanded panel support from 1 to 8
models, replaced the static `STATE_TO_COMMAND` dict with a
panel-aware, capability-gated `CommandResolver`, hit three live
production regressions on our own SDVECU and hotfixed each, added an
armed → armed UI guard mirroring the mobile app, and shipped v0.9.0 to
PyPI + GitHub with a git-history scrub to remove a leaked numinst.

### Approach — reverse-engineering, not speculation

Started from issue #3 (SDVFAST panel, interior-only variant). The
earlier path — asking reporters to run commands and see what fails —
was replaced by web-bundle decompilation: `customers.verisure.it`
serves a public JS bundle containing both the panel roster and the
resolver function. `scripts/dissect-web-bundle.sh` auto-detects the
latest bundle version, downloads it, and prints the panel enum,
ArmCodeRequest / DisarmCodeRequest values, and the decoded target →
command lookup. Committed to the repo so every release cycle can
catch upstream API drift before users hit it.

### Features shipped (11 subagent-driven tasks + 3 hotfixes + UI guard)

- **`SUPPORTED_PANELS` expanded** to all 8 models from the bundle:
  SDVECU, SDVECUD, SDVECUW, SDVECU-D, SDVECU-W, MODPRO (peri-capable
  family); SDVFAST, SDVFSW (interior-only family). `PANEL_FAMILIES`
  encodes the classification.
- **`CommandResolver`** — panel + current-state + active-services
  aware. Lives in `verisure_italy/resolver.py`. Replaces target-only
  `STATE_TO_COMMAND` dict (removed). Handles peri-capable vs
  interior-only families, single-step armed transitions
  (`ARMINTFPART1`, `ARMPARTFINTDAY1`), and reliable sub-capability
  gates (`ARMNIGHT`, `ARMANNEX`).
- **Cross-perimeter armed-transition guard** — raises `ValueError`
  on `TOTAL_PERI → PARTIAL` (perimeter OFF) and similar; old resolver
  would have silently emitted `ARMDAY1`. Fail-secure safety addition.
- **`ArmCommand` wire vocabulary** — full set (`ARMINTFPART1`,
  `ARMPARTFINTDAY1`, `ARMPARTFINTNIGHT1`, `ARMNIGHT1`, `ARMANNEX1`,
  `DARMANNEX1`, `ARMINTEXT1`, `DARMPERI`). `ServiceRequest` enum +
  `active_services()` helper. `UnsupportedCommandError` raised
  pre-HTTP when service gate blocks.
- **`StateNotObservedError`** — raised when arm/disarm runs before
  first `xSStatus` observation (integration reload race). Surfaces as
  a visible HA notification instead of an unhandled traceback. Must
  subclass `VerisureError` — the HA exception handler only catches
  that base class. Caught in code review.
- **Structured failure reports** — every arm/disarm failure emits an
  ERROR-level log wrapped in `=== VERISURE ARM FAILURE BEGIN ===` /
  `=== VERISURE ARM FAILURE END ===` markers (or `DISARM`). Contains
  panel, family, hashed numinst, current proto, selected command,
  active services, exception type/code/message. HA notification text
  points user at the block and the GH issues URL — PII-safe paste
  workflow.
- **`diagnostics.py`** — merged `probe.py` in; adds
  `format_failure_report()`. Single diagnostic module.
- **HA entity layer short-circuits** — redundant arm/disarm at
  current state no longer pays a wire round-trip.
- **Services cache** — `xSSrv` fetched once per session per
  installation, invalidated on capabilities-token rotation.

### Live testing — three production bugs hotfixed on our SDVECU

1. **Capability gate too strict** (`b47dd87`). Initial
   `_COMMAND_REQUIRES` mapped perimeter + day variants to specific
   `ServiceRequest` flags. Real SDVECU `xSSrv` does NOT list `ARMDAY`
   or `PERI` as active services, yet the panel accepts `ARMDAY1` /
   `ARM1PERI1` / `ARMDAY1PERI1` / `DARM1DARMPERI` fine. `ARMDAY`/`PERI`
   rows in the service list are UI hints, not wire-protocol gates.
   Fixed by gating only on base `ARM`/`DARM` + reliably-reported
   sub-caps (`ARMNIGHT`, `ARMANNEX`, `DARMANNEX`), plus
   family-based peri gate (`INTERIOR_ONLY` rejects `*PERI*`).
2. **Transition commands reject on SDVECU** (`4e626c6`). Resolver
   tried `ARMPARTFINTDAY1` for armed → armed transitions on SDVECU,
   panel refused with "Request not valid for Central Unit". SDVFAST
   probe shows `ARMINTFPART` / `ARMPARTFINT` as active services;
   SDVECU doesn't. Fixed by gating the transition commands on those
   sub-capabilities — SDVECU now falls through to target-only
   `ARM_TOTAL` / `ARM_PARTIAL` (matches pre-v0.9.0 behaviour that
   worked in production for months).
3. **Armed → armed cross-perimeter**: panel rejects
   `ARMDAY1PERI1` with `error_code 106` / `error_mpj_exception` when
   already in `TOTAL_PERI`. Chose a UI + code guard over
   orchestration.

### Armed → armed UI guard

User chose KISS over implementing disarm-first orchestration. Mirrors
the Verisure mobile app: arm buttons gray out while armed.

- `_update_supported_features()` computes `_attr_supported_features`
  from current alarm state. Disarmed → `ARM_HOME | ARM_AWAY`; any
  armed state → `0`. Cached `supported_features` invalidated via
  `vars(self).pop(...)` on each state update (pyright disallows
  `@property` override of HA's `cached_property` base; `_attr_`
  + manual cache invalidation is the working pattern).
- `_async_arm` refuses armed → armed with a WARN log — belt-and-braces
  against direct service calls from automations or REST.
- Verified live: armed_away panel, REST `alarm_arm_home` call
  rejected by HA's service dispatcher with `ServiceNotSupported`
  (no code-layer reach, no wire traffic).

Saved as feedback memory: prefer mirroring upstream UX constraints
over writing orchestration.

### Decisions

- **Family-based peri gate, not `PERI` service flag** —
  `PERI` / `ARMDAY` are UI hints in `xSSrv`, not wire gates.
  INTERIOR_ONLY family classification gates perimeter commands
  instead. Documented in `docs/findings/arm-command-vocabulary.md`.
- **UI hides arm buttons while armed, not client-side disarm-then-arm
  orchestration** — KISS, mirrors Verisure mobile app UX, avoids a
  disarmed window (security matters).
- **Web-bundle dissection over reporter probing** — public static JS,
  no auth required. Committed as `scripts/dissect-web-bundle.sh` for
  ongoing drift detection.
- **Hashed numinst + full history scrub** — even though numinst alone
  isn't a credential, user treats it as PII. Force-push of rewritten
  public history is acceptable for solo repos.
